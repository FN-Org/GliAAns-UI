## Section Details
#### 1. Requirements Engineering

Documentation related to the analysis and design phase of the project.

[URS](./urs/urs.md) (User Requirement Specifications): Describes what the system should do from the user's perspective. These define the business needs and user expectations.

[DRS](./drs/drs.md) (Design Requirement Specifications): Contains the technical system requirements, architectural design, and detailed specifications necessary for developers to implement the solution.

#### 2. Manuals & Guides

Practical guides for installing, building, and executing the software. You can find the following guides in the manuals folder:

| File	                               | Description                                                                             | 
|:------------------------------------|:----------------------------------------------------------------------------------------|
| [install.md](./manuals/install.md)	 | Step-by-step instructions on how to configure the environment and install dependencies. |
| [compile.md](./manuals/compile.md)	 | Instructions on how to build the source code from scratch.                              |
| [run.md](./manuals/run.md)	         | Guidelines on how to launch the application and basic usage examples.                   |

#### 3.References

[ref](./ref): This folder contains all external resources, academic papers, documentation in medical field, q&a with the stakeholders and other materials consulted during the development of this project.
